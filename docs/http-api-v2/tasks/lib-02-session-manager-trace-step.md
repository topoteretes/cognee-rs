### 2.1 Persisted-payload shape vs request-payload shape — clarification (2026-04-29 investigation)

Python distinguishes the **request** payload (`TraceEntry`, used to call `add_agent_trace_step`) from the **persisted** payload (`SessionAgentTraceEntry`, what actually gets serialized into the cache). The Rust port must mirror the persisted shape because that is what the detail endpoint (E-12) reads back.

Persisted fields per [`SessionAgentTraceEntry`](https://github.com/topoteretes/cognee/blob/main/cognee/infrastructure/databases/cache/models.py#L88-L112):

```
trace_id: str          (server-set, returned by add_agent_trace_step)
origin_function: str
status: str            ("success" | "error", validator only enforces non-empty)
memory_query: str = ""
memory_context: str = ""
method_params: dict     (default {} — note: NOT Optional in persisted shape)
method_return_value: Any = None
error_message: str = ""
session_feedback: str = ""   (LLM-generated by SessionManager BEFORE persisting; not in TraceEntry)
```

Notes on what is **NOT** in the persisted payload:
- **No `created_at`**: Python's persisted entry carries no timestamp. Order is implicit in the list (oldest-first). E-12's wire shape similarly has no per-step timestamp. The Rust port should drop `created_at` from `SessionTraceStep` to keep parity.
- **No `user_id` / `session_id`**: those live in the cache key (`agent_sessions:{user_id}:{session_id}` on Redis; equivalent on diskcache). The Rust port may carry them on the type for ergonomic callsite use, but they are not serialized into the entry body — handle via `serde(skip)` or do not emit them.
- **No `generate_feedback_with_llm`**: that flag belongs to the request payload (`TraceEntry`), not the persisted entry. `SessionManager` reads it, then either calls the LLM or uses a fallback to compute `session_feedback`, then persists the resolved `session_feedback` string.

Trace step request payload fields (mirror `TraceEntry`):

```
origin_function: str
status: "success" | "error"
method_params: Optional[dict]
method_return_value: Optional[Any]
memory_query: str
memory_context: str
error_message: str
generate_feedback_with_llm: bool       (default False per TraceEntry; True per add_agent_trace_step kwarg)
trace_id: uuid                         (server-set, returned)
```

- `cognee-test-utils` does **not** export a central `MockSessionStore`. The only mock impl is inline in [`crates/delete/src/lib.rs:3565-3659`](../../../crates/delete/src/lib.rs#L3565) (delete-tests-only, tracks `pruned()` and stubs every method). The doc's "default impl returns `SessionError::Unsupported`" requires either adding a new `Unsupported` variant or reusing `StoreError("not implemented".into())` — see step 2 below.

## 4. Implementation steps

1. **Add `SessionTraceStep` type** in `crates/session/src/types.rs` — match Python's persisted `SessionAgentTraceEntry` (no `created_at`, no `generate_feedback_with_llm`, includes `session_feedback`):
   ```rust
   #[derive(Debug, Clone, Serialize, Deserialize)]
   pub struct SessionTraceStep {
       pub trace_id: String,
       pub origin_function: String,
       pub status: String,                // free-form per Python validator (typically "success" / "error")
       #[serde(default)]
       pub memory_query: String,
       #[serde(default)]
       pub memory_context: String,
       #[serde(default)]
       pub method_params: serde_json::Value,    // default `{}` (Python uses `Dict[str, Any]` with `default_factory=dict`)
       #[serde(default)]
       pub method_return_value: Option<serde_json::Value>,
       #[serde(default)]
       pub error_message: String,
       #[serde(default)]
       pub session_feedback: String,
   }
   ```
   `serde` field naming stays `snake_case` (Python persists snake_case in the cache; this is library-internal data, not a wire DTO — Decision 10 / camelCase only applies to HTTP DTOs).

   Notes vs the original draft:
   - **No `TraceStatus` enum** — Python's validator only enforces non-empty string; agent trace steps may carry application-defined status strings beyond `"success"` / `"error"`. Keep the field as `String` for forward compat; if both backends ever standardise on a closed set, tighten later.
   - **No `created_at`** — Python's `SessionAgentTraceEntry` has no timestamp; ordering is positional (oldest-first append). Removing it keeps the persisted JSON byte-equal between SDKs.
   - **No `user_id` / `session_id` on the type** — those are the cache key, not part of the entry body. The `save_trace_step` signature carries them as separate args (see step 2).

2. **Extend `SessionStore` trait** (`crates/session/src/session_store.rs`). Use separate `user_id` / `session_id` args on writes — same pattern as the existing `create_qa_entry`:
   ```rust
   /// Append one agent-trace step to the session's trace list.
   /// Returns the persisted `trace_id` (caller-provided — `SessionManager` generates it).
   async fn save_trace_step(
       &self,
       user_id: &str,
       session_id: &str,
       step: SessionTraceStep,
   ) -> Result<String, SessionError> {
       let _ = (user_id, session_id, step);
       Err(SessionError::StoreError(
           "save_trace_step not implemented for this backend".into(),
       ))
   }

   /// Retrieve agent-trace steps for the given session (oldest-first).
   /// Caller (`SessionManager::get_agent_trace_session`) handles `last_n` slicing.
   async fn read_trace_steps(
       &self,
       user_id: &str,
       session_id: &str,
   ) -> Result<Vec<SessionTraceStep>, SessionError> {
       let _ = (user_id, session_id);
       Err(SessionError::StoreError(
           "read_trace_steps not implemented for this backend".into(),
       ))
   }
   ```
   Default impls reuse `SessionError::StoreError` (no new variant needed — keeps `error.rs` untouched). Each backend overrides with a real impl below; `MockSessionStore` in `crates/delete/src/lib.rs` inherits the default and continues to compile.

3. **Implement on each backend**:
   - **`fs_store.rs`** — adjacent JSON file `{base_dir}/{user_id}/{session_id}.traces.json` with the same load/save helpers as the existing `session_file()`. Use a parallel `trace_session_file()` so the QA file stays untouched. Order: append on write, return list verbatim on read.
   - **`redis_store.rs`** — `RPUSH cognee:trace:{user_id}:{session_id}` (Python uses `RPUSH`, NOT `LPUSH`; `LRANGE 0 -1` returns oldest-first naturally — see [RedisAdapter.py:470](https://github.com/topoteretes/cognee/blob/main/cognee/infrastructure/databases/cache/redis/RedisAdapter.py#L470)). The original draft saying "LPUSH" was wrong; corrected here.
   - **`sea_orm_store.rs`** — new SeaORM entity `SessionTraceStepEntity` in `sea_orm_backend/entity.rs` (table `session_trace_steps`, PK `trace_id`; columns `user_id`, `session_id`, `origin_function`, `status`, `memory_query`, `memory_context`, `method_params` (Text/JSON), `method_return_value` (Text/JSON nullable), `error_message`, `session_feedback`, `seq` BIGINT auto-increment for stable insert ordering). Index on `(user_id, session_id, seq)`. Read sorted by `seq ASC`. Add a migration `m20260429_000003_session_trace_steps.rs` in `crates/session/src/migrator/` and register it in `migrator/mod.rs`.

4. **Add `SessionManager` wrappers** in `crates/session/src/session_manager.rs`:
   ```rust
   /// Append one agent trace step. Returns the generated `trace_id`.
   pub async fn add_agent_trace_step(
       &self,
       user_id: &str,
       session_id: Option<&str>,
       origin_function: &str,
       status: &str,
       memory_query: &str,
       memory_context: &str,
       method_params: serde_json::Value,
       method_return_value: Option<serde_json::Value>,
       error_message: &str,
       session_feedback: &str,
   ) -> Result<String, SessionError>;

   /// Retrieve agent trace steps for a session (oldest-first). `last_n` tail-truncates.
   pub async fn get_agent_trace_session(
       &self,
       user_id: &str,
       session_id: Option<&str>,
       last_n: Option<usize>,
   ) -> Result<Vec<SessionTraceStep>, SessionError>;
   ```
   - `add_agent_trace_step`: server-generates `trace_id = uuid::Uuid::new_v4().to_string()` (Python uses UUID4 — see [session_manager.py:284](https://github.com/topoteretes/cognee/blob/main/cognee/infrastructure/session/session_manager.py#L284)). Builds a `SessionTraceStep` and calls `store.save_trace_step(user_id, resolved_session_id, step)`. Returns the `trace_id`.
   - `get_agent_trace_session`: forwards to `store.read_trace_steps`, then (if `last_n` is `Some(n)`) takes the trailing `n` entries with `entries.split_off(entries.len().saturating_sub(n))`. Mirrors Python `entries[-last_n:]` ([session_manager.py:621](https://github.com/topoteretes/cognee/blob/main/cognee/infrastructure/session/session_manager.py#L621)).
   - Note: Python's `add_agent_trace_step` also runs an LLM-feedback generation step before persisting (see [session_manager.py:285-297](https://github.com/topoteretes/cognee/blob/main/cognee/infrastructure/session/session_manager.py#L285-L297)). The Rust port keeps `session_feedback` as a **caller-supplied parameter** in this task — LLM feedback generation is owned by LIB-01 (the `remember_entry()` facade). Until LIB-01 lands, callers pass `""` for `session_feedback`; the persisted shape stays compatible.

5. **Mock impl in `crates/delete/src/lib.rs`** — verify it still compiles. The default trait impls cover `save_trace_step` / `read_trace_steps`; no edit required unless a delete test exercises trace pruning (none does at present — `delete_session` does not currently iterate traces, and Python's `delete_session` is the only place that touches both QA + trace data; cross-cutting cleanup is out of scope here and will surface in a later task if needed).

## 5. Tests

Follow the existing per-backend test split (`crates/session/tests/fs_store_tests.rs`, `crates/session/tests/sea_orm_store_tests.rs`) — extend each with trace-step coverage, plus a new shared file for the `SessionManager` wrapper:

- `crates/session/tests/fs_store_tests.rs` — add:
  - `test_fs_save_and_read_trace_step` — round-trip a single `SessionTraceStep`; assert all fields preserved byte-equal.
  - `test_fs_trace_steps_oldest_first` — three sequential saves, `read_trace_steps` returns them in insertion order.
  - `test_fs_trace_step_isolation` — two `(user_id, session_id)` pairs do not see each other's entries.
- `crates/session/tests/sea_orm_store_tests.rs` — add the same three cases against the SeaORM backend (in-memory SQLite). Migration must run before each test (the existing `setup` helper handles this — confirm the new migration is registered).
- `crates/session/tests/test_session_manager_trace.rs` (new):
  - `test_add_agent_trace_step_returns_trace_id` — `add_agent_trace_step` returns a non-empty UUID-shaped string.
  - `test_trace_step_uuid_uniqueness` — 100 inserts, 100 distinct `trace_id`s.
  - `test_get_agent_trace_session_last_n` — insert 5, request `last_n=3`, get last 3 in oldest-first order. Mirrors Python `entries[-last_n:]`.
  - `test_get_agent_trace_session_default_returns_all` — `last_n=None` returns everything.
  - `test_unimplemented_backend_returns_store_error` — a stub `SessionStore` that does not override the new methods returns `SessionError::StoreError`.

Skip the redis test path for now — no redis test file exists; gate the `redis_store.rs` impl on the `redis` feature and add a redis-only test only if/when redis test infrastructure lands (out of scope here).
