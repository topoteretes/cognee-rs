# Phase 7 ÔÇö TypeScript layer & existing-code actualization

ÔćÉ [Index](../typescript-bindings-plan.md)

**Goal:** wrap the native functions in an idiomatic, fully-typed TypeScript SDK, and **bring the
existing `js/` code current** ÔÇö re-home the legacy engine API under `cognee.pipeline.*`, rewrite
the entry points, and settle the package identity. This phase satisfies the "existing code is
modified and actualized" goal.

## Scope

- **In:** the `Cognee` class, the TypeScript type surface, the legacy re-homing, package
  identity, and the file-by-file actualization of everything already in `js/`.
- **Out:** new native functionality (Phases 1ÔÇô6) and error internals (Phase 8).

## `js/src/cognee.ts` ÔÇö the `Cognee` class (NEW file)

### Constructor
```ts
constructor(settings?: object | string)
```
Calls `native.cogneeNew(settings)` synchronously to create the opaque handle. Stores the box
internally as `this._handle: NativeBox` (private; not exposed in the public API).

### Public async methods

All methods follow this exact pattern ÔÇö wrapping the native call in `try/catch` and re-throwing
via `wrapNativeError`:
```ts
async add(
  dataInput: CogneeDataInput | CogneeDataInput[],
  datasetName: string,
  opts?: CogneeAddOptions
): Promise<CogneeAddResult> {
  try {
    return await native.cogneeAdd(this._handle, dataInput, datasetName, opts);
  } catch (e) {
    throw wrapNativeError(e);
  }
}
```

Full method list with signatures:
- `add(dataInput, datasetName, opts?)` Ôćĺ `CogneeAddResult`
- `cognify(dataset, opts?)` Ôćĺ `CogneeCognifyResult`
- `addAndCognify(dataInput, datasetName, opts?)` Ôćĺ `{ add: CogneeAddResult; cognify: CogneeCognifyResult }`
- `search(query, opts?)` Ôćĺ `CogneeSearchResponse`
- `recall(query, opts?)` Ôćĺ `CogneeRecallResult`
- `remember(dataInput, datasetName, opts?)` Ôćĺ `CogneeRememberResult`
- `rememberEntry(entry, datasetName, sessionId, opts?)` Ôćĺ `CogneeRememberResult`
- `memify(opts?)` Ôćĺ `CogneeMemifyResult`
- `improve(opts)` Ôćĺ `CogneeImproveResult`
- `forget(target, opts?)` Ôćĺ `CogneeForgetResult`
- `update(dataId, newData, datasetName, opts?)` Ôćĺ `CogneeUpdateResult`
- `pruneData()` Ôćĺ `void`
- `pruneSystem(opts?)` Ôćĺ `CogneePruneResult`
- `visualize(opts?)` Ôćĺ `string` (HTML string)
- `visualizeToFile(opts?)` Ôćĺ `string` (absolute path)
- `warm()` Ôćĺ `void`
- `ownerId()` Ôćĺ `string`

### `serve` and `disconnect` ÔÇö module-level, not instance methods

`cogneeServe` and `cogneeDisconnect` in the native layer take **no handle** ÔÇö they operate on
global cloud state. In the TypeScript layer, these are exposed as **module-level exported
functions from `cognee.ts`** (not methods on `Cognee`):
```ts
export async function serve(opts?: CogneeServeOptions): Promise<CogneeServeResult> {
  try {
    return await native.cogneeServe(opts);
  } catch (e) {
    throw wrapNativeError(e);
  }
}

export async function disconnect(opts?: CogneeDisconnectOptions): Promise<void> {
  try {
    await native.cogneeDisconnect(opts);
  } catch (e) {
    throw wrapNativeError(e);
  }
}
```
Both are also re-exported from `index.ts`.

### `cognee.config` sub-object (inline implementation)

`config` is a **plain object literal** created in the constructor body and assigned to
`this.config`. Each method is an arrow function that closes over `this._handle`:

```ts
this.config = {
  setLlmProvider:  (v: string)  => { native.configSetLlmProvider(this._handle, v); },
  setLlmModel:     (v: string)  => { native.configSetLlmModel(this._handle, v); },
  setLlmApiKey:    (v: string)  => { native.configSetLlmApiKey(this._handle, v); },
  // ÔÇŽ all other configSet* methods ÔÇŽ
  set:             (key: string, value: unknown) => {
    try { native.configSet(this._handle, key, value); }
    catch (e) { throw wrapNativeError(e); }
  },
  get: (): Record<string, unknown> => native.getConfig(this._handle),
};
```

Bulk setters (`setLlmConfig`, `setEmbeddingConfig`, `setVectorDbConfig`, `setGraphDbConfig`) also
throw via `wrapNativeError` because the native layer can throw `UNKNOWN_CONFIG_KEY` /
`CONFIG_TYPE_MISMATCH`.

All **granular** `configSet*` setters are sync and do not need try/catch (they can only fail on
programming error, not runtime conditions). **Do not wrap them in try/catch.**

TypeScript type for `config`:
```ts
config: {
  setLlmProvider(v: string): void;
  setLlmModel(v: string): void;
  // ÔÇŽ full list matching NativeBindings configSet* methods ÔÇŽ
  set(key: string, value: unknown): void;
  get(): Record<string, unknown>;
}
```

### `cognee.datasets` sub-object (inline implementation)

Same pattern ÔÇö plain object literal, arrow functions closing over `this._handle`:
```ts
this.datasets = {
  list:         () => native.cogneeListDatasets(this._handle),
  listData:     (datasetId: string) => native.cogneeListData(this._handle, datasetId),
  has:          (datasetId: string) => native.cogneeHasData(this._handle, datasetId),
  status:       (datasetIds: string[]) => native.cogneeDatasetStatus(this._handle, datasetIds),
  empty:        (datasetId: string) => native.cogneeEmptyDataset(this._handle, datasetId),
  deleteData:   (datasetId, dataId, opts?) => native.cogneeDeleteData(this._handle, datasetId, dataId, opts),
  deleteAll:    () => native.cogneeDeleteAllDatasets(this._handle),
};
```
All are async (return Promises from native); no need for try/catch unless a method can throw
synchronously ÔÇö these all return Promises, so errors surface naturally via rejection.

## `wrapNativeError` usage ÔÇö per-method pattern

Every async method on `Cognee` that calls native must use this exact pattern. No exceptions:

```ts
async methodName(ÔÇŽ): Promise<ReturnType> {
  try {
    return await native.cogneeXxx(this._handle, ÔÇŽ);
  } catch (e) {
    throw wrapNativeError(e);
  }
}
```

The `wrapNativeError` function is imported from `./errors` inside `cognee.ts`. It already
exists in `js/src/errors.ts` ÔÇö do not create a new implementation.

For sync methods (all `configSet*` granular setters), no try/catch is added.
For fallible sync methods (`configSet`, `configSetLlmConfig`, etc.), use:
```ts
try { native.configSet(this._handle, key, value); }
catch (e) { throw wrapNativeError(e); }
```

## Risks

- The `index.ts` re-org is **breaking for `@cognee/pipeline` importers** ÔÇö document the
  migration (`import { Pipeline }` Ôćĺ `import { pipeline } from "cognee"`).
- **`pipeline.ts` collision:** the existing file contains the `Pipeline` class with that exact
  name. Option A (keep `Pipeline` in `pipeline.ts`, add re-exports at the bottom) is chosen.
  This avoids any rename or move.
- **`serve`/`disconnect` have no handle:** they operate on global cloud state. They are
  module-level functions, not `Cognee` instance methods.
- Keep TS types in lockstep with the serde shapes; a mismatch is silent until runtime.
- `native.ts` grows large ÔÇö the `types.ts` split keeps `native.ts` focused on the binding
  contract (`NativeBox`, `NativeBindings`, `native`).
