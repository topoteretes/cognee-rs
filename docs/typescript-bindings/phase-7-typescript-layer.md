# Phase 7 — TypeScript layer & existing-code actualization

← [Index](../typescript-bindings-plan.md)

**Goal:** wrap the native functions in an idiomatic, fully-typed TypeScript SDK, and **bring the
existing `js/` code current** — re-home the legacy engine API under `cognee.pipeline.*`, rewrite
the entry points, and settle the package identity. This phase satisfies the "existing code is
modified and actualized" goal.

## Scope

- **In:** the `Cognee` class, the TypeScript type surface, the legacy re-homing, package
  identity, and the file-by-file actualization of everything already in `js/`.
- **Out:** new native functionality (Phases 1–6) and error internals (Phase 8).

## Structures (new / rewritten TS)

### `js/src/cognee.ts` — the `Cognee` class
- `constructor(settings?)` → `cogneeNew`.
- Methods, all returning typed Promises: `add`, `cognify`, `addAndCognify`, `search`, `recall`,
  `remember`, `rememberEntry`, `memify`, `forget`, `update`, `prune`, `improve`, `visualize`,
  `serve`, `disconnect`.
- Sub-objects for grouped surfaces: `cognee.config` (setters + `get()`), `cognee.datasets`
  (list/has/status/empty/delete), `cognee.sessions` (get/feedback/graphContext).
- `cognee.pipeline` — the legacy engine namespace (below).

### `js/src/types.ts` — the type surface
- Input unions: `DataInput`, `MemoryEntry`, `ForgetTarget`, `DatasetRef`, `PruneTarget`,
  `RecallScope`.
- Enums/unions: `SearchType` (exact serde strings).
- Result types: `AddResult`, `CognifyResult`, `SearchResponse`, `RecallResult`, `RememberResult`,
  `MemifyResult`, `ForgetResult`, `UpdateResult`, `PruneResult`, `ImproveResult`, `Dataset`,
  `Data`, `SessionQAEntry`.
- Options objects per method.

### `js/src/pipeline.ts` — legacy aggregator
- Re-export the existing engine classes (`Pipeline`, `Task`/`createTask*`, `Watcher`,
  `ProgressToken`, `CancellationHandle`/`Token`, `RunHandle`, `RunResult`, `TaskContext`,
  `CogneeValue`) so they live under `cognee.pipeline.*`.

## Existing-code actualization (Goal 1) — file by file

| Existing file | Action |
|---|---|
| `js/package.json` | **Decide identity:** name is `@cognee/pipeline` (wrong for a full SDK). Rename (e.g. `@cognee/cognee` / `cognee`) or re-scope; fix `description`, `main`, `types`, `files`, build scripts. |
| `js/src/index.ts` | Export the `Cognee` class + types as the default surface; `export * as pipeline from "./pipeline"`. Add a migration note (breaking re-org). |
| `js/src/native.ts` | Declare every new native function (handle, config, ops, feature-gated). |
| `js/src/*.ts` (pipeline, task, watcher, progress, cancellation, run-handle, run-result, task-context, value) | Keep; re-export through `pipeline.ts`. Otherwise unchanged. |
| `js/__tests__/*` | Keep engine tests (update imports to the `pipeline` namespace); SDK tests added in Phase 9. |
| `js/lib/*` | Generated by `tsc` — regenerated, not hand-edited; confirm ignore/refresh policy. |
| `js/README.md` | Rewrite around the SDK quick start; demote engine docs to a "low-level pipeline API" appendix. |

## Functionalities

- Ergonomic, discoverable API: `const c = new Cognee({...}); await c.add(...); await c.cognify();
  await c.search("…")`.
- Config either via constructor settings or `c.config.setLlmModel(...)`.
- Full IntelliSense from `.d.ts` (generated by `tsc` from `types.ts`).

## Dependencies & ordering

Needs the native functions from Phases 1–6. Best done incrementally alongside each op phase
(add the typed method as each native function lands), with the final consolidation here.

## Risks

- The `index.ts` re-org is **breaking** for any current `@cognee/pipeline` consumer — document
  the migration (`import { Pipeline }` → `import { pipeline } from "cognee"`).
- Keep TS types in lockstep with the serde shapes; a mismatch is silent until runtime.

## Done when

- `import { Cognee } from "<pkg>"` exposes the full typed SDK; legacy is reachable under
  `.pipeline`.
- Every existing `js/` file is either updated or intentionally re-exported; README rewritten.
